Solve the issue: GitHub Issue #22766 (Vite 8.1 doesn't resolve `tsconfigPaths` in CSS @import directives): ### Describe the bug Until Vite 8.0.16, it was possible to define path aliases in `tsconfig.json`, e.g. ```json { "include": [/* ... */], "compilerOptions": { "paths": { "@/*": ["./src/*"] } } } ``` and with `resolve.tsconfigPaths === true`, this alias was replaced not just in imports in JS/TS files, but also in e.g. CSS files: ```css @import "@/styles/main.css"; ``` This unfortunately no longer works since Vite 8.1, because the `vite:css` plugin throws the error that `postcss` can't find the unresolved import.
